Convert natural language questions to SPARQL queries for an Ontotext GraphDB containing FHIR resources.  
The output must include a correctly formatted SPARQL query using the standard prefixes.  
Do not output any explanatory text outside of the SPARQL query block.

---

## CLASS AND PROPERTIES LIST
Patient[deceased_datetime, gender, age, race, ethnicity, organization_reference, id, identifier]
MedicationDispense[medicationRequest_reference, encounter_reference, patient_reference, dosageInstruction_reference, id, identifier, status, medication_name]
DosageInstruction[doseQuantity_code, doseQuantity_unit, doseQuantity_value, maxDosePerPeriod_denominator_unit, maxDosePerPeriod_denominator_value, maxDosePerPeriod_numerator_value, medication_frequency_code, timing_repeat_duration, timing_repeat_duration_unit]

---

##Example SPARQL QUERY 

PREFIX fhir: <http://hl7.org/fhir/>
PREFIX rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX se: <http://example.org/myontology#>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

SELECT DISTINCT ?patientId ?medicationDispenseID ?medication_name ?status
WHERE {
  ?medicationDispense a fhir:MedicationDispense .
  ?medicationDispense fhir:patient_reference ?patient .

  ?patient a fhir:Patient .

  # Optional Patient fields
    OPTIONAL { ?patient fhir:deceased_datetime ?deceased_datetime. }
    OPTIONAL { ?patient fhir:gender ?gender. }
    OPTIONAL { ?patient fhir:age ?age. }
    OPTIONAL { ?patient fhir:race ?race. }
    OPTIONAL { ?patient fhir:ethnicity ?ethnicity. }
    OPTIONAL { ?patient fhir:id ?patientId. }
    OPTIONAL { ?patient fhir:identifier ?patient_identifier. }
    OPTIONAL { ?patient fhir:organization_reference ?organization. }

  # Optional MedicationDispense fields
  OPTIONAL { ?medicationDispense fhir:medicationRequest_reference ?medicationRequest. }
  OPTIONAL { ?medicationDispense fhir:encounter_reference ?encounter_reference. }
  OPTIONAL { ?medicationDispense fhir:dosageInstruction_reference ?dosageInstruction. }
  OPTIONAL { ?medicationDispense fhir:id ?medicationDispenseId. }
  OPTIONAL { ?medicationDispense fhir:identifier ?identifier. }
  OPTIONAL { ?medicationDispense fhir:status ?status. }
  OPTIONAL { ?medicationDispense fhir:medication_name ?medication_name. }

  # Optional dosageInstruction fields
    OPTIONAL {
    ?medicationDispense fhir:dosageInstruction_reference ?dosageInstruction.
    ?dosageInstruction a fhir:DosageInstruction .
      # Optional DosageInstruction fields
      OPTIONAL { ?dosageInstruction fhir:doseQuantity_code ?doseQuantity_code. }
      OPTIONAL { ?dosageInstruction fhir:doseQuantity_unit ?doseQuantity_unit. }
      OPTIONAL { ?dosageInstruction fhir:doseQuantity_value ?doseQuantity_value. }
      OPTIONAL { ?dosageInstruction fhir:maxDosePerPeriod_denominator_unit ?maxDosePerPeriod_denominator_unit. }
      OPTIONAL { ?dosageInstruction fhir:maxDosePerPeriod_denominator_value ?maxDosePerPeriod_denominator_value. }
      OPTIONAL { ?dosageInstruction fhir:maxDosePerPeriod_numerator_value ?maxDosePerPeriod_numerator_value. }
      OPTIONAL { ?dosageInstruction fhir:medication_frequency_code ?medication_frequency_code. }
      OPTIONAL { ?dosageInstruction fhir:timing_repeat_duration ?timing_repeat_duration. }
      OPTIONAL { ?dosageInstruction fhir:timing_repeat_duration_unit ?timing_repeat_duration_unit. }
    }


}

-Always filter using FILTER(CONTAINS(LCASE(STR(?medication_name)), "your-display-text-here") ) 
-Keep as many fields optional as possible while still working as a query
